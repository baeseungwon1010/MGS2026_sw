도메인이란

도메인은 인터넷 상에서 식별되는 웹사이트나 네트워크의 주소

트리 구조로

오른쪽부터 최상위 도메인→ 왼쪽으로 하위 도메인

!image.png

도메인 탐색 방법(간략하게)

1. DNS 조회: 직접 dns 서버에 질의하여 존재 유무를 확인
2. CT log 조회: 공개된 CT(certificate transparency:ssl 인증서 로그) 로그 등에서 ssl/tls 인증서에 기록된 서브도메인을 수집
3. dns 레코드 분석: dns 레코드를 분석하여 다른 도메인이나 관련 서브도메인 발견
4. bruteforce: 워드리스트를 이용하거나 bruteforce를 활용하여 dns 질의 수행
5. whois 조회: whois 정보를 이용하여 등록 기관, 네임 서버, 관련 도메인 등의 정보 수집(서브도메인은 그다지 효력이 나쁨)
6. 외부 데이터 소스(OSINT): 공개 데이터베이스와 검색 엔진에서 서브도메인을 수집한다(dns 질의 안함)

버그바운티 연계

osint 기반 탐색 → 실제 dns 조회 → httpx 등 웹서비스 확인→ nuclei 등 취약점 스캔

다른 사람이 관리해야할 도메인의 제어권을 휙득하는 공격

만료된 도메인을 제 3자가 재등록

dns 관리 계정 탈취

dns 레코드에서 연결된 일부 삭제된 서브도메인에서 dns가 남아 있을 때 서비스를 생성하여 이를 자신의 서버에 연결→ 공격 가능

dns 레코드 정리

| 레코드 | 역할 | 예시 |
| --- | --- | --- |
| A | 도메인을 IPv4에 연결 | example.com → 93.184.216.34 |
| AAAA | 도메인을 IPv6에 연결 | example.com → 2606:2800:220::: |
| CNAME | 도메인을 다른 도메인의 별칭으로 지정 | api.example.com → api.azurewebsites.net |
| MX | 메일 서버 지정 | example.com → mail.protection.outlook.com |
| NS | 해당 도메인을 관리하는 dns 서버 지정 | ns1.cloudflare.com |
| TXT | 문자열 정보 저장 | v=spf1 include:_spf.google.com ~all |
| SOA | 도메인의 기본 DNS 정보 | zone 관리 정보 |
| PTR | IP 주소를 도메인으로 변환 | 8.8.8.8 → dns.google |
| SRV | 특정 서비스의 서버 위치와 포트 지정 | sip, ldap 등에서 사용 |
| CAA | SSL/TLS 인증서를 발급할 수 있는 인증기관(CA) 지정 | letsencrypt.org만 허용 |
| NAPTR | sip, voip 등의 서비스 탐색 | _sip._tcp.example.com → sipserver.example.com |
| SSHFP | ssh 서버의 공개키 지문 저장 | server.example.com → SHA-256 Fingerprint |
| TLSA | DANE를 위해 tls 인증서 정보를 저장 | _443._tcp.example.com → TLS Certificate Hash |
| DS | 상위 도메인과 하위 도메인의 dnssec 신뢰 체인 연결 | .com → example.com DS |
| DNSKEY | dnssec에서 사용하는 공개키 저장 | example.com → DNSSEC Public Key |
| HINFO | 서버의 cpu와 운영체제 정보를 저장 | server.example.com → Intel Xeon, Linux |
| ISDN | isdn 회선의 전화번호 저장 | office.example.com → 0312345678 |
| RRSIG | dnssec에서 dns 레코드의 전자서명, 정보를 저장하여 무결성 검증 | example.com A → Digital Signature |

DNSSEC: dns 응답이 위조되지 않았음을 검증하는 보안 기술

dns 응답과 전자서명을 같이 전달

dns resolver는 dnskey를 이용해 이를 검증

dns zone마다 고유한 dnskey를 가짐, 레코드 집합 마다 전자서명이 붙음
